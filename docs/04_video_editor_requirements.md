# 04. 영상 편집 기능 요구사항

> [!IMPORTANT]
> 경기 영상 편집기 v1의 상세하고 우선하는 기준은 `docs/31_video_editor_v1_requirements.md`다.
> 이 문서는 편집 기능의 목적과 상위 원칙을 요약한다.
> 두 문서가 충돌하면 31번 요구사항을 적용한다.

## 1. 기능 목적

영상 편집기는 플랫폼의 핵심 경쟁력이다.

단순히 영상을 자르는 기능이 아니라 COACH와 ANALYST가 경기 장면을 빠르게 찾고, 전술적 의도를 시각적으로 설명하며, 선수에게 완성된 결과를 공유하는 분석 도구로 설계한다.

---

## 2. 사용자와 환경

### COACH·ANALYST

- 최신 Chrome·Edge 기반 PC 편집
- 영상 도메인 동일한 전체 권한
- 하나의 경기 작업공간에서 팀·선수 클립 제작

### PLAYER

- 모바일 중심 결과 조회
- READY MP4만 시청
- 편집 화면과 미완성 상태 접근 불가

---

## 3. 작업공간

하나의 경기 원본 영상 화면에서 다음을 제공한다.

- 원본 영상 재생
- 현재 경기 클립 목록
- 새 팀 분석 클립
- 새 선수 개인 분석 클립
- DRAFT·FAILED 편집
- 개별·일괄 렌더링
- 상태 확인

상태 그룹:

```text
DRAFT → 작업 중
QUEUED → 생성 대기
PROCESSING → 생성 중
READY → 완료
FAILED → 실패
```

---

## 4. 구간 편집

- 하나의 클립은 하나의 연속 구간
- 원본 구간 최소 1초
- 최종 결과 최대 15분
- I/O 시작·종료 설정
- 타임라인 핸들
- 시간 직접 입력
- 1초·5초 이동
- 원본 FPS 기반 한 프레임 이동
- 반복 재생과 미리보기
- 시간은 밀리초 저장

비연속 장면은 별도 클립으로 만든다.

---

## 5. 분석 요소

### 드로잉

- 선
- 화살표
- 원
- 박스
- 영역
- 텍스트
- 선수 강조
- 패스·수비 라인
- 압박 구역

좌표는 영상 표시 영역 기준 정규화 좌표로 저장한다.

### 화면 정지

- 기본 3초
- 최소 1초, 최대 10초
- 정지 구간 오디오 무음

### 줌

- 기본 1.5배
- 1.2배 이상 3배 이하
- 중복 줌 구간 금지
- 드로잉과 동시 사용 가능

v1에서는 복잡한 카메라 팬과 다중 트랙을 제공하지 않는다.

---

## 6. 저장과 복구

- 변경 후 1~2초 debounce 자동 저장
- 편집 중 30초 안전 저장
- IndexedDB 로컬 복구
- 10분 단위 서버 스냅샷
- 수동 저장, 렌더링 전, 강제 잠금 해제 전 스냅샷
- 클립당 최신 20개
- READY 완료 시 스냅샷 정리
- 최종 편집 원본은 READY 이후에도 유지

---

## 7. 편집 잠금

- 클립 단위 잠금
- 같은 클립은 한 명만 편집
- 다른 사용자는 읽기 전용
- 30초 heartbeat
- 10분 미응답 시 만료
- 잠금 토큰과 편집 버전 사용
- 오래된 저장은 409 Conflict
- COACH·ANALYST 강제 해제 가능

초기에는 MySQL을 사용하되 향후 Redis로 교체할 수 있도록 서비스 계층을 분리한다.

---

## 8. 렌더링

- DRAFT·FAILED 렌더링 가능
- 공용 FIFO 대기열
- 동시 실행 기본 2개
- 일괄 최대 20개
- 각 클립 독립 처리
- 실제 PROCESSING 시작 후 60분 타임아웃
- 서버 재시작 시 PROCESSING은 FAILED
- 자동 재시도 없음

결과:

```text
MP4
H.264(libx264)
AAC
yuv420p
CRF 21
preset medium
최대 1080p
최대 60fps
faststart
```

드로잉, 텍스트, 정지와 줌을 결과 MP4에 합성한다.

---

## 9. READY 정책

- PLAYER 권한 범위에 따라 즉시 조회
- 제목·유형·코멘트만 직접 수정
- 구간·대상 선수·분석 효과는 직접 수정 불가
- 영상 내용을 변경하려면 편집 원본을 복사해 새 DRAFT 생성
- 원본 영상이 삭제됐거나 파일이 없으면 새 DRAFT 생성 불가

---

## 10. 썸네일

- 결과 MP4 1초 지점
- 실패 시 중간 지점
- 다시 실패하면 최초 디코딩 프레임
- 전부 실패하면 기본 썸네일
- 썸네일 실패만으로 READY를 취소하지 않음

---

## 11. 영상 보안

- 운영 환경에서 `/uploads/**` 직접 공개 금지
- 인증 스트리밍 또는 Signed URL
- HTTP Range 지원
- 개인 클립 대상 선수 검증
- 내부 파일 경로와 FFmpeg 상세 오류 비노출

---

## 12. 삭제

### 클립

DRAFT, FAILED와 READY는 즉시 영구 삭제한다.

QUEUED와 PROCESSING은 삭제할 수 없다.

### 원본 영상

- 소프트 삭제
- 10일 복구
- 10일 후 원본 MP4 영구 삭제
- 원본 북마크와 팀 클립은 복구하지 않음
- READY 개인 클립 유지
- 미완성 개인 클립은 10일 후 삭제

---

## 13. 제외 범위

- 비연속 구간 합치기
- 다중 트랙
- 배경음악과 음성 녹음
- 복잡한 전환
- 모바일 편집
- 자동 선수 추적
- AI 이벤트 인식
- 자동 하이라이트
- CDN 실제 운영
