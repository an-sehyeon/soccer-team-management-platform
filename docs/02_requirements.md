# 02. 요구사항 정리

## 1. 프로젝트 목적

COACH, ANALYST와 PLAYER가 함께 사용하는 실제 축구팀 운영 및 경기 영상분석 플랫폼을 개발한다.

기능 구현뿐 아니라 권한, 보안, 영상 비용, 장애 복구, 사용성과 유지보수를 고려한다.

---

## 2. 사용자 요구사항

### COACH

- 로그인과 팀 운영
- 스케줄·공지사항 CRUD
- 경기 원본 영상 등록·조회·수정·삭제·복구
- 팀·선수 개인 분석 클립 제작
- 드로잉·텍스트·정지·줌 편집
- DRAFT 저장과 복구
- 렌더링 요청·재시도
- 선수 기록 등록·수정과 영상 연결

### ANALYST

- 로그인
- 스케줄·공지사항 조회
- 영상 도메인에서 COACH와 동일한 전체 관리
- 원본 영상 등록·수정·삭제·복구
- 분석 클립 편집·렌더링·삭제
- 선수 기록 등록과 클립 연결

### PLAYER

- 로그인
- 스케줄·공지사항 조회
- 접근 가능한 원본 영상 조회
- READY 팀 분석 클립 조회
- 본인의 READY 선수 개인 분석 클립 조회
- 본인 기록과 이벤트 조회

---

## 3. 기능 요구사항

### 3.1 인증과 권한

- JWT 인증을 사용한다.
- 권한은 백엔드에서 검증한다.
- `isAdmin`만으로 영상 관리 권한을 부여하지 않는다.
- 개인 분석 영상은 대상 선수 본인과 COACH·ANALYST만 접근한다.
- PLAYER에게 편집 상태 클립을 반환하지 않는다.

### 3.2 스케줄과 공지사항

- COACH는 등록·수정·삭제·조회한다.
- ANALYST와 PLAYER는 조회한다.

### 3.3 경기 원본 영상

- COACH와 ANALYST가 실제 MP4를 업로드한다.
- ffprobe로 길이, 해상도, FPS와 오디오 정보를 추출한다.
- 모든 역할은 허용 범위에서 원본 영상을 조회한다.
- 삭제는 10일 복구 가능한 소프트 삭제다.
- QUEUED 또는 PROCESSING 클립이 있으면 삭제할 수 없다.

### 3.4 경기 영상 편집기

상세 기준은 `docs/31_video_editor_v1_requirements.md`를 따른다.

- PC 전용
- 하나의 경기 작업공간
- 여러 팀·선수 클립 생성
- 하나의 클립은 하나의 연속 구간
- 원본 구간 최소 1초
- 최종 결과 최대 15분
- 밀리초 시간 저장
- 원본 FPS 기반 프레임 이동
- 드로잉·텍스트·정지·줌
- 자동 저장과 복구
- 클립 단위 편집 잠금
- 개별·일괄 렌더링

### 3.5 클립 상태

```text
DRAFT
QUEUED
PROCESSING
READY
FAILED
```

- 신규 클립은 DRAFT
- DRAFT·FAILED는 렌더링 가능
- QUEUED·PROCESSING은 편집·삭제 불가
- READY는 완성 MP4 재생
- PLAYER는 READY만 조회

### 3.6 렌더링

- 팀·선수 공용 FIFO 대기열
- 동시 실행 기본 2개
- 일괄 최대 20개
- 60분 타임아웃
- 서버 재시작 시 PROCESSING은 FAILED
- H.264 + AAC MP4
- 최대 1080p·60fps
- CRF 21, preset medium
- 드로잉·정지·줌을 결과 MP4에 합성
- 결과 썸네일 자동 생성

### 3.7 분석 클립 삭제

DRAFT, FAILED, READY는 즉시 영구 삭제한다.

- MP4와 썸네일
- 편집 데이터
- 북마크와 조회 기록
- 이벤트 연결

선수 기록 이벤트와 누적 통계는 유지한다.

### 3.8 영상 북마크

- COACH·ANALYST 본인 북마크
- PLAYER 접근 불가
- READY 영상만 지원
- 밀리초 시간
- 클립 제작 초기값으로 활용
- 직접 클립 삭제 시 관련 북마크 영구 삭제

### 3.9 선수 기록

- COACH·ANALYST가 경기 영상 기준으로 등록
- PLAYER는 본인 기록 조회
- READY 분석 클립만 이벤트와 연결
- 직접 클립 삭제 시 연결만 삭제
- 원본 삭제로 팀 클립이 소프트 삭제되면 연결 이력 유지
- 클립 삭제로 통계를 자동 차감하지 않음

## 4. 비기능 요구사항

### 보안

- 영상 파일 경로를 공개하지 않는다.
- 운영 환경의 `/uploads/** permitAll`을 제거한다.
- 인증 스트리밍 또는 Signed URL을 사용한다.
- HTTP Range 요청을 지원한다.
- 서버 경로와 FFmpeg 명령어를 API에 노출하지 않는다.

### 사용성

- PLAYER 화면은 모바일 우선
- 편집기는 PC 우선
- 저장·잠금·렌더링 상태를 명확히 표시
- 삭제는 복구 가능 여부를 명확히 안내
- 사용자에게 해결 가능한 오류만 표시

### 성능

- 렌더링 동시 실행 수를 제한한다.
- 설정값을 외부화한다.
- 대용량 파일과 JSON 크기를 제한한다.
- 목록은 페이지네이션한다.

### 정합성

- DB 트랜잭션과 파일 정리를 분리해 재시도한다.
- 오래된 편집 버전 저장을 차단한다.
- READY 전 결과 파일을 검증한다.
- 서버 재시작 후 상태를 복구한다.

### 확장성

- 저장소 인터페이스 분리
- 잠금 서비스 분리
- 렌더링 서비스 분리
- 추후 Redis, 작업 큐, S3와 CDN으로 교체 가능
- AI 자동 분석은 v1 범위에서 제외
