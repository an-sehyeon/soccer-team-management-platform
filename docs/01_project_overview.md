# 01. 프로젝트 개요

## 1. 프로젝트 목적

이 프로젝트는 실제 축구팀에 판매하고 운영할 목적으로 개발하는 팀 운영 및 경기 영상분석 플랫폼이다.

단순 기능 구현보다 다음을 우선한다.

- 실제 팀 업무에 사용할 수 있는 편집 품질
- 역할과 개인 영상 접근 제어
- 안정적인 영상 저장·재생
- 파일과 DB 정합성
- 지도자·분석관의 PC 작업 효율
- 선수의 모바일 조회 편의성
- 운영 비용과 유지보수성

---

## 2. 플랫폼 개요

플랫폼은 일정, 공지사항, 경기 원본 영상, 분석 클립, 드로잉, 북마크, 선수 기록과 조회 이력을 하나의 서비스에서 관리한다.

핵심 경쟁력은 경기 영상 편집기다.

지도자와 분석관은 하나의 경기 영상 작업공간에서 팀 분석 클립과 선수 개인 분석 클립을 제작한다. 드로잉, 텍스트, 화면 정지와 줌 효과는 최종 MP4에 합성한다.

선수는 모바일에서 READY 상태의 완성된 분석 영상을 시청하고 본인 기록을 확인한다.

---

## 3. 주요 사용자

### COACH

- 팀 일정과 공지사항 관리
- 경기 영상 전체 관리
- 팀·선수 분석 클립 제작
- 편집·렌더링·삭제·복구
- 선수 기록 관리

### ANALYST

- 일정과 공지사항 조회
- 영상 도메인에서 COACH와 동일한 전체 관리
- 경기 영상 업로드와 편집
- 팀·선수 분석 클립 제작
- 렌더링과 분석 데이터 관리
- 선수 기록 등록과 클립 연결

### PLAYER

- 스케줄과 공지사항 조회
- 접근 가능한 원본 영상 조회
- READY 팀 분석 영상 조회
- 본인의 READY 선수 개인 분석 영상 조회
- 본인 기록과 이벤트 조회

## 5. 경기 영상 편집기 방향

경기 영상 편집기 v1의 상세 기준은 다음 문서를 따른다.

```text
docs/31_video_editor_v1_requirements.md
```

주요 방향:

- 편집기는 PC 전용
- 하나의 경기 작업공간에서 여러 클립 제작
- 팀·선수 개인 클립을 같은 화면에서 관리
- 클립과 편집 시간은 밀리초
- 원본 FPS 기반 프레임 이동
- DRAFT, QUEUED, PROCESSING, READY, FAILED 상태
- 최종 MP4에 분석 요소 합성
- PLAYER는 READY 결과만 조회
- 인증과 권한이 적용된 영상 재생

## 6. 서비스 사용 환경

### 선수

- 모바일 우선
- 빠른 목록 조회
- 안정적인 MP4 재생
- 본인 개인 영상과 기록 접근성 우선

### 지도자·분석관

- PC 우선
- 큰 영상 플레이어와 타임라인
- 키보드 기반 구간 탐색
- 드로잉과 효과 편집
- 여러 클립 상태 관리
- 일괄 렌더링

---

## 7. 기술 방향

### Backend

Java, Spring Boot, Spring Security, JPA, MySQL, FFmpeg와 ffprobe를 사용한다.

### Frontend

React와 TypeScript를 사용한다.

### 영상 저장

초기에는 로컬 파일 저장소를 사용하되 저장소 계층을 분리해 추후 오브젝트 스토리지와 CDN으로 확장한다.

DB에는 파일 자체를 저장하지 않는다.

### 보안

운영 환경에서 `/uploads/**`를 공개하지 않는다. 인증 스트리밍 또는 Signed URL 방식으로 전환한다.
