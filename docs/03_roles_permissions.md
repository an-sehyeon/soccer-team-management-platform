# 역할 및 권한 정책

## 역할 구분

이 플랫폼의 사용자는 크게 지도자와 선수로 구분한다.

지도자는 팀 운영과 경기 영상 분석을 관리하는 사용자이다.

선수는 지도자가 등록한 일정, 공지사항, 분석 영상, 기록을 확인하는 사용자이다.

## 지도자 권한

지도자는 다음 기능을 사용할 수 있다.

### 스케줄

* 스케줄 등록
* 스케줄 수정
* 스케줄 삭제
* 스케줄 조회

### 공지사항

* 공지사항 등록
* 공지사항 수정
* 공지사항 삭제
* 공지사항 조회

### 경기 영상

* 경기 영상 업로드
* 경기 영상 조회
* 경기 영상 수정
* 경기 영상 삭제
* 경기 영상 편집

### 팀 분석 영상

* 팀 분석 클립 생성
* 팀 분석 클립 수정
* 팀 분석 클립 삭제
* 팀 분석 클립 조회

### 선수 개인 분석 영상

* 선수 개인 분석 클립 생성
* 선수 개인 분석 클립 수정
* 선수 개인 분석 클립 삭제
* 선수 개인 분석 클립 전체 조회

### 선수 기록

* 선수별 기록 조회
* 선수별 기록 등록
* 선수별 기록 수정

## 선수 권한

선수는 다음 기능을 사용할 수 있다.

### 스케줄

* 스케줄 조회

### 공지사항

* 공지사항 조회

### 팀 분석 영상

* 팀 분석 클립 조회

### 선수 개인 분석 영상

* 본인 개인 분석 클립 조회

### 선수 기록

* 본인 기록 조회

## 영상 접근 정책

영상 접근 권한은 이 플랫폼에서 매우 중요하다.

특히 선수 개인 분석 영상은 반드시 접근 제어가 필요하다.

### 팀 분석 영상

팀 분석 영상은 팀 전체에게 공유되는 영상이다.

접근 가능한 사용자:

* 해당 팀 지도자
* 해당 팀 소속 선수

### 선수 개인 분석 영상

선수 개인 분석 영상은 특정 선수에게만 공유되는 영상이다.

접근 가능한 사용자:

* 해당 팀 지도자
* 해당 영상의 대상 선수 본인

접근 불가능한 사용자:

* 같은 팀의 다른 선수
* 다른 팀 지도자
* 다른 팀 선수
* 로그인하지 않은 사용자

## 권한 검증 기준

권한 검증은 반드시 백엔드에서 처리한다.

프론트에서 버튼을 숨기는 것만으로는 보안이 되지 않는다.

예를 들어 선수 화면에서 영상 편집 버튼을 숨기더라도, 선수가 API 주소를 직접 호출할 수 있기 때문에 백엔드에서 다시 권한을 확인해야 한다.

## 권한 설계 시 주의사항

* 선수 개인 영상은 본인만 볼 수 있어야 한다.
* 지도자는 자기 팀 영상만 볼 수 있어야 한다.
* 다른 팀의 영상에는 접근할 수 없어야 한다.
* 추후 분석관, 코치, 감독처럼 지도자 역할이 세분화될 수 있으므로 역할 확장을 고려한다.
* 처음부터 너무 복잡하게 만들지는 않되, 확장이 막히지 않도록 설계한다.
