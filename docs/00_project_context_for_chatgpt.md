# 00. ChatGPT 프로젝트 컨텍스트

## 1. 문서 목적

이 문서는 축구팀 영상분석 플랫폼의 새 채팅을 시작할 때 사용하는 최신 요약 문서다.

상세 요구사항과 실제 코드는 GitHub를 기준으로 확인한다.

```text
Repository:
https://github.com/an-sehyeon/soccer-team-management-platform

Docs:
https://github.com/an-sehyeon/soccer-team-management-platform/tree/main/docs
```

현재 사용자가 로컬과 GitHub `main`의 상태가 동일하다고 명시했다. 필요한 실제 코드, DTO, Entity, API 주소와 기존 요구사항은 GitHub `main`을 직접 확인한다. 이미 `main`에서 확인 가능한 파일을 사용자에게 다시 업로드해 달라고 요청하지 않는다.

현재 브랜치에서 새로 변경한 내용은 해당 브랜치 소스를 기준으로 확인한다.

## 3. 프로젝트 목적

이 프로젝트는 포트폴리오 예제가 아니라 실제 축구팀에 판매하고 운영할 플랫폼이다.

핵심 기준:

- 권한과 개인 영상 접근 제어
- 안정적인 영상 업로드·편집·재생
- 데이터와 파일 정합성
- 지도자·분석관의 PC 작업 효율
- 선수의 모바일 조회 편의성
- 운영 비용과 장애 복구
- 유지보수와 확장성

---

## 5. 사용자 역할

### COACH

팀 운영과 영상 도메인의 전체 관리 권한을 가진다.

### ANALYST

스케줄과 공지사항은 조회만 가능하다.

영상 도메인에서는 COACH와 동일한 전체 권한을 가진다.

- 원본 영상 등록·수정·삭제·복구
- 팀·선수 클립 생성·수정·삭제
- 편집·렌더링·재시도
- DRAFT·스냅샷·잠금 관리
- 선수 기록 등록과 클립 연결

### PLAYER

- 스케줄·공지사항 조회
- 접근 가능한 원본 영상 조회
- READY 팀 클립 조회
- 본인의 READY 개인 클립 조회
- 본인 기록과 이벤트 조회

편집 상태와 관리 기능에는 접근할 수 없다.

`isAdmin`만으로 영상 또는 선수 기록 관리 권한을 부여하지 않는다.

## 7. 경기 영상 편집기 v1 확정 정책

상세 문서:

```text
docs/31_video_editor_v1_requirements.md
```

핵심:

- 하나의 경기 작업공간에서 팀·선수 클립 생성
- PC 전용 편집
- 클립 시간과 편집 시간 밀리초
- 원본 FPS 기반 프레임 이동
- 드로잉·텍스트·정지·줌
- DRAFT 자동 저장과 IndexedDB 복구
- 클립 단위 편집 잠금
- 상태: DRAFT, QUEUED, PROCESSING, READY, FAILED
- 팀·선수 공용 FIFO 렌더링 대기열
- 동시 렌더링 기본 2개
- 일괄 렌더링 최대 20개
- 최대 1080p·60fps
- H.264 + AAC MP4
- CRF 21, preset medium
- 결과 썸네일 자동 생성
- PLAYER는 READY만 조회
- 인증 기반 Range 스트리밍
- 최종 편집 데이터는 READY 이후에도 유지
- 클립은 즉시 영구 삭제
- 원본 영상은 10일 복구 가능한 소프트 삭제

---

## 8. 삭제 정책 요약

### 분석 클립

DRAFT, FAILED, READY는 즉시 영구 삭제한다.

삭제 대상:

- 클립 행
- MP4와 썸네일
- 편집 데이터
- 드로잉·정지·줌
- 북마크와 조회 기록
- 선수 기록 이벤트 연결
- 스냅샷과 임시 파일

선수 기록 이벤트 자체와 누적 기록은 유지한다.

QUEUED와 PROCESSING은 삭제할 수 없다.

### 경기 원본 영상

- 소프트 삭제
- 원본 MP4 10일 유지
- 10일 내 복구 가능
- 원본 북마크와 팀 클립은 복구하지 않음
- 팀 클립 MP4 즉시 삭제
- READY 개인 클립 유지
- 10일 후 원본 MP4와 DRAFT·FAILED 개인 클립 영구 삭제
- 삭제 메타데이터는 유지

---

## 9. 선수 기록과 북마크 연동

### 선수 기록

- READY 클립만 이벤트 연결 가능
- 직접 클립 영구 삭제 시 연결 정보만 삭제
- 선수 기록 이벤트와 통계는 유지
- 원본 삭제로 팀 클립이 소프트 삭제되면 연결 이력 유지
- 화면에 `해당 클립이 삭제되었습니다` 표시
- 신규 canonical 이벤트 시간은 밀리초

### 북마크

- COACH·ANALYST 본인 데이터
- PLAYER 접근 불가
- READY 영상만 지원
- 신규 canonical 시간은 밀리초
- 직접 클립 삭제 시 북마크 영구 삭제
- 원본 삭제 시 원본·팀 클립 북마크는 복구하지 않음

## 11. 새 채팅 진행 규칙

새 채팅에서는 다음 순서로 진행한다.

1. GitHub `main`의 문서와 실제 소스 확인
2. 현재 구현 구조 요약
3. 31번 요구사항과 충돌하는 코드·DB 목록 작성
4. 변경할 Entity, Enum, SQL과 DTO 호환 방향 제안
5. 데이터 마이그레이션과 회귀 위험 정리
6. 구현 순서 제안
7. 사용자가 진행한다고 하면 코드 작성

GitHub `main`에 있는 파일을 다시 요청하지 않는다.

정확한 이름은 실제 소스에서 확인하고 추측하지 않는다.

사업 정책에 영향을 주는 중요한 미확정 사항만 질문한다. 구현 세부사항은 운영 안정성을 기준으로 최선의 안을 선택한다.

코드 수정 시 기존 파일 구조를 기준으로 변경 사항이 반영된 전체 파일 코드를 제공한다.
