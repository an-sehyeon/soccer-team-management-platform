# 29. 경기 영상 북마크 기능 요구사항

> [!IMPORTANT]
> 경기 영상 편집기 v1 이후 시간 단위, READY 불변성, 삭제와 영상 제공 정책은
> `docs/31_video_editor_v1_requirements.md`를 우선한다.
> 아래 0번 갱신 정책이 기존 본문의 충돌하는 내용을 대체한다.

## 0. 경기 영상 편집기 v1 연동 갱신 정책

### 0.1 북마크 시간 단위

신규 북마크 canonical 시간은 밀리초다.

```text
bookmark_time_ms
```

- 0 이상
- 대상 원본 또는 결과 영상 길이 이하
- 같은 밀리초에 복수 북마크 허용
- 기존 `bookmark_time_sec` 데이터는 `기존 값 × 1000`으로 마이그레이션

정확한 컬럼명과 호환 DTO 유지 기간은 실제 Entity·SQL과 프론트 호출부를 확인한 뒤 확정한다. 신규 편집기 내부 상태는 밀리초를 사용한다.

### 0.2 지원 상태

북마크는 재생 가능한 READY 영상에서만 사용한다.

```text
경기 원본 READY
팀 분석 클립 READY
선수 개인 분석 클립 READY
```

DRAFT, QUEUED, PROCESSING과 FAILED 클립에는 북마크를 등록하거나 조회하지 않는다.

### 0.3 READY 클립 불변성

경기 영상 편집기 v1에서 READY 클립의 구간과 분석 효과는 직접 수정하지 않는다.

영상 내용을 바꾸려면 기존 편집 원본으로 새 DRAFT를 만든다.

따라서 기존 본문의 “READY 클립 시작·종료 시간 변경 시 북마크 소프트 삭제” 정책은 신규 편집 흐름에서는 발생하지 않는다. 기존 호환 API가 남아 있는 동안에만 방어 로직으로 유지한다.

### 0.4 직접 클립 영구 삭제

팀 또는 선수 개인 분석 클립을 직접 영구 삭제하면 해당 클립의 북마크도 영구 삭제한다.

복구 목록이나 삭제 이력을 제공하지 않는다.

### 0.5 원본 경기 영상 삭제

원본 경기 영상 삭제 시:

- 원본 북마크는 소프트 삭제
- 복구하지 않음
- 원본 삭제로 소프트 삭제되는 팀 클립의 북마크도 소프트 삭제
- 팀 클립을 복구하지 않으므로 북마크도 복구하지 않음
- READY 선수 개인 클립의 북마크는 해당 READY 클립이 유지되는 동안 유지

### 0.6 영상 제공

기존 `/uploads/**` 직접 URL은 MVP 호환 구조다.

경기 영상 편집기 v1에서는 인증 스트리밍 또는 짧은 만료 재생 URL을 사용하며, 프론트가 서버 내부 파일 경로를 직접 조합하지 않는다.

### 0.7 분석 작업 초기값

북마크에서 새 클립을 시작할 때:

```text
DRAFT 시작 시간 = bookmark_time_ms
```

종료 시간은 클립 최대 15분 정책과 원본 잔여 길이를 고려해 편집기에서 유효한 범위로 초기화한다. 사용자가 최종 구간을 직접 조정한다.

## 1. 문서 목적

이 문서는 경기 원본 영상, 팀 분석 클립, 선수 개인 분석 클립에서 지도자와 분석관이 개인 북마크를 저장하고 관리하는 기능의 최신 정책을 정의한다.

북마크는 공유 콘텐츠가 아니라 로그인 사용자 개인의 임시 분석 작업 목록이다.

---

## 2. 기능 목적

- 경기 영상에서 다시 확인할 장면을 빠르게 저장한다.
- 영상별 개인 작업 목록을 만든다.
- 북마크 장면을 팀 분석 클립과 선수 개인 분석 클립 생성의 시작점으로 활용한다.
- 경기 영상을 다시 탐색하는 시간을 줄인다.
- 지도자와 분석관의 PC 영상 분석 작업 효율을 높인다.

북마크는 선수에게 공유되는 콘텐츠가 아니다.

---

## 3. 사용자 역할

### COACH

- 본인 북마크 등록
- 본인 북마크 목록 조회
- 본인 북마크 수정
- 본인 북마크 삭제
- 북마크를 기준으로 분석 작업 시작

### ANALYST

- 본인 북마크 등록
- 본인 북마크 목록 조회
- 본인 북마크 수정
- 본인 북마크 삭제
- 북마크를 기준으로 분석 작업 시작

북마크는 개인 임시 데이터이므로 분석관도 본인 북마크를 삭제할 수 있다.

### PLAYER

- 북마크 API 접근 불가
- 북마크 UI 미노출

## 4. 접근 정책

- 사용자는 본인이 작성한 북마크만 조회할 수 있다.
- 사용자는 본인이 작성한 북마크만 수정할 수 있다.
- 사용자는 본인이 작성한 북마크만 삭제할 수 있다.
- 다른 사용자의 북마크 ID를 직접 호출해도 접근이 거부되어야 한다.
- 권한과 소유자 검증은 반드시 백엔드에서 처리한다.

## 5. 지원 영상 소스

북마크는 다음 영상 소스를 지원한다.

```text
경기 원본 영상
팀 분석 클립
선수 개인 분석 클립
```

DB에서 소스는 별도 `source_type` 컬럼 없이 FK 조합으로 구분한다.

### 경기 원본 영상 북마크

```text
upload_id 존재
team_clip_id null
player_clip_id null
```

### 팀 분석 클립 북마크

```text
upload_id 존재
team_clip_id 존재
player_clip_id null
```

### 선수 개인 분석 클립 북마크

```text
upload_id 존재
team_clip_id null
player_clip_id 존재
```

두 클립 ID가 동시에 존재하는 데이터는 허용하지 않는다.

---

## 6. 북마크 데이터

주요 필드는 다음과 같다.

```text
id
upload_id
team_clip_id
player_clip_id
member_id
bookmark_time_sec
title
memo
is_deleted
created_at
updated_at
```

### 시간

- 정수 초 단위로 저장한다.
- 0 이상이어야 한다.
- 현재 영상 길이를 초과할 수 없다.
- 같은 시간에 여러 북마크를 등록할 수 있다.

### 제목

- 필수값이다.
- 앞뒤 공백을 제거한다.
- 공백만 입력할 수 없다.
- 최대 255자다.

### 메모

- 선택값이다.
- 앞뒤 공백을 제거한다.
- 공백만 입력하면 `null`로 저장한다.
- 최대 255자다.

---

## 7. 영상 상태 정책

북마크는 재생 가능한 `READY` 상태 영상에서만 등록·조회·수정할 수 있다.

- 경기 원본 영상이 삭제된 경우 조회하지 않는다.
- 팀 분석 클립이 삭제되거나 `READY`가 아닌 경우 조회하지 않는다.
- 선수 개인 분석 클립이 삭제되거나 `READY`가 아닌 경우 조회하지 않는다.

---

## 8. 클립 변경과 북마크 삭제 정책

팀 분석 클립 또는 선수 개인 분석 클립의 시작·종료 시간이 변경되면 해당 클립 내부 시간 기준으로 등록된 북마크의 의미가 달라진다.

따라서 다음 경우 해당 클립의 활성 북마크를 모두 소프트 삭제한다.

- 팀 분석 클립 시작 시간 변경
- 팀 분석 클립 종료 시간 변경
- 팀 분석 클립 삭제
- 선수 개인 분석 클립 시작 시간 변경
- 선수 개인 분석 클립 종료 시간 변경
- 선수 개인 분석 클립 원본 경기 영상 변경
- 선수 개인 분석 클립 삭제

다음 변경은 북마크를 유지한다.

- 제목 변경
- 코멘트 변경
- 클립 유형 변경
- 대상 선수 변경
- 드로잉 추가·수정

사용자 안내 문구는 다음 취지를 사용한다.

```text
기존 클립이 수정 또는 삭제되어 해당 클립에서 생성한 북마크가 모두 삭제되었습니다.
```

## 9. API

### 등록

```http
POST /api/management/video-bookmarks
```

요청 예시는 다음과 같다.

```json
{
  "matchVideoId": 1,
  "teamClipId": null,
  "playerClipId": null,
  "bookmarkTimeSec": 125,
  "title": "전환 수비 확인",
  "memo": "수비 간격 확인 필요"
}
```

### 목록 조회

```http
GET /api/management/video-bookmarks?matchVideoId={matchVideoId}
GET /api/management/video-bookmarks?matchVideoId={matchVideoId}&teamClipId={teamClipId}
GET /api/management/video-bookmarks?matchVideoId={matchVideoId}&playerClipId={playerClipId}
```

목록은 최신 생성 순으로 반환한다.

### 수정

```http
PATCH /api/management/video-bookmarks/{bookmarkId}
```

수정 가능 항목은 다음과 같다.

```text
bookmarkTimeSec
title
memo
```

### 삭제

```http
DELETE /api/management/video-bookmarks/{bookmarkId}
```

삭제는 소프트 삭제다.

## 10. 백엔드 처리 흐름

### 등록

1. JWT 인증 정보를 확인한다.
2. `COACH` 또는 `ANALYST`인지 검증한다.
3. 영상 소스 FK 조합을 검증한다.
4. 경기 원본 영상이 존재하고 삭제되지 않았는지 확인한다.
5. 클립 소스이면 클립이 존재하고 삭제되지 않았는지 확인한다.
6. 대상 영상 또는 클립이 `READY` 상태인지 확인한다.
7. 요청 경기 영상과 클립 원본 경기 영상이 일치하는지 확인한다.
8. 북마크 시간이 영상 길이 안에 있는지 검증한다.
9. 제목과 메모를 정리한다.
10. 로그인 사용자 소유 북마크로 저장한다.

### 목록 조회

1. 로그인 사용자 ID를 기준으로 조회한다.
2. 요청한 현재 영상 소스와 일치하는 북마크만 조회한다.
3. 삭제된 북마크를 제외한다.
4. 삭제되거나 재생 불가능한 소스의 북마크를 제외한다.
5. 최신 생성 순으로 반환한다.

### 수정·삭제

1. 북마크 존재 여부를 확인한다.
2. 로그인 사용자 소유인지 확인한다.
3. 소스가 여전히 유효한지 확인한다.
4. 수정 또는 소프트 삭제한다.

## 11. 프론트 화면 정책

### 11.1 북마크 버튼

- `COACH`, `ANALYST`에게만 표시한다.
- `READY` 영상에서만 활성화한다.
- 버튼명은 `북마크 목록`을 사용한다.

### 11.2 사이드바

북마크 목록은 페이지 아래 영역이 아니라 화면 오른쪽 고정 사이드바로 표시한다.

- `position: fixed` 기반 오른쪽 사이드바
- PC에서는 반투명 배경 표시
- 외부 배경 클릭 시 닫기
- `Esc` 키로 닫기
- 닫기 버튼 제공
- 모바일에서는 전체 화면 너비 사용
- 사이드바 내부 세로 스크롤

### 11.3 북마크 등록

사이드바에서 현재 영상 시간을 기준으로 등록한다.

```text
현재 시점
제목
메모
저장 버튼
```

현재 시간은 소수점 이하를 버리고 정수 초로 저장한다.

### 11.4 북마크 선택

북마크를 클릭하면 다음과 같이 동작한다.

1. 현재 영상을 일시 정지한다.
2. 선택한 북마크 시간으로 이동한다.
3. 자동 재생하지 않는다.
4. 사이드바를 닫는다.
5. 화면을 영상 플레이어 위치로 이동한다.
6. 영상 아래에 선택한 북마크 정보를 표시한다.

이전 정책의 북마크 선택 즉시 자동 재생은 사용하지 않는다.

### 11.5 선택 북마크 정보

영상 아래에 다음을 표시한다.

```text
시간
제목
메모
```

분석 작업 버튼은 다음을 제공한다.

```text
팀 분석 클립 등록
선수 개인 분석 클립 등록
선수 기록 등록
```

## 12. 북마크와 분석 클립 생성

### 경기 원본 영상 북마크

```text
분석 시작 시간 = bookmark_time_sec
분석 종료 시간 = 원본 경기 영상 duration_sec
```

팀 분석 클립과 선수 개인 분석 클립 생성 패널에 초기값으로 전달한다.

사용자는 이후 시작·종료 시간을 직접 조정할 수 있다.

### 팀 분석 클립 내부 북마크

```text
원본 경기 영상 시작 시간
= team_video_clip.start_time_sec + bookmark_time_sec

원본 경기 영상 종료 시간
= team_video_clip.end_time_sec
```

### 선수 개인 분석 클립 내부 북마크

```text
원본 경기 영상 시작 시간
= player_video_clip.start_time_sec + bookmark_time_sec

원본 경기 영상 종료 시간
= player_video_clip.end_time_sec
```

실제 분석 패널로 이동할 때는 원본 경기 영상 기준 시간으로 변환한다.

## 13. 북마크와 선수 기록 등록

`선수 기록 등록`은 북마크 시간을 선수 기록 이벤트 시간으로 사용하지 않는다.

최신 선수 기록 정책은 다음과 같다.

### 클립 없이 등록

- 북마크 시간 사용 안 함
- `player_record` 요약 기록만 생성 또는 갱신
- 이벤트 시작·종료 시간 없음

### 팀 분석 클립 연결

- 선택한 팀 분석 클립의 시작·종료 시간을 백엔드가 사용
- 북마크 시간 사용 안 함

### 선수 개인 분석 클립 연결

- 선택한 선수 개인 분석 클립의 시작·종료 시간을 백엔드가 사용
- 북마크 시간 사용 안 함

따라서 선택 북마크 영역의 `선수 기록 등록` 버튼은 선수 기록 등록 패널 진입 기능만 수행한다.

북마크 시간은 선수 기록 요청 DTO로 전달하지 않는다.

## 14. 프론트 타입

```ts
export type CreateVideoBookmarkRequest = {
  matchVideoId: number;
  teamClipId: number | null;
  playerClipId: number | null;
  bookmarkTimeSec: number;
  title: string;
  memo: string | null;
};
```

## 16. DB 설계 방향

`video_bookmark` 테이블을 사용한다.

삭제는 소프트 삭제다.

주요 인덱스 방향은 다음과 같다.

```text
member_id + upload_id + is_deleted
member_id + team_clip_id + is_deleted
member_id + player_clip_id + is_deleted
```

서비스 계층에서 소유자와 소스 유효성을 함께 검증한다.
